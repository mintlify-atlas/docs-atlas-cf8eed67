---
title: Environment Variables
description: Configure fzf behavior using environment variables for default commands and options
---

fzf can be configured using environment variables that control its default behavior. These variables are particularly useful for setting consistent defaults across all your fzf invocations.

## Core Environment Variables

<ParamField path="FZF_DEFAULT_COMMAND" type="string">
  Default command to use when input is tty (not piped). This command generates the list of candidates when fzf is run without input.

  <Note>
    Without STDIN pipe, fzf will traverse the file system under the current directory to get the list of files. You can override this behavior by setting `FZF_DEFAULT_COMMAND`.
  </Note>

  **Example: Using fd for file listing**
  ```bash
  export FZF_DEFAULT_COMMAND='fd --type f'
  ```

  **Example: Respecting .gitignore**
  ```bash
  export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix'
  ```

  **Example: Including hidden files and following symlinks**
  ```bash
  export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix --hidden --follow --exclude .git'
  ```

  **Example: Using ripgrep for file listing**
  ```bash
  export FZF_DEFAULT_COMMAND='rg --files --hidden --follow --glob "!.git/*"'
  ```
</ParamField>

<ParamField path="FZF_DEFAULT_OPTS" type="string">
  Default options applied to all fzf invocations. These options are parsed and merged with command-line options.

  **Example: Basic UI customization**
  ```bash
  export FZF_DEFAULT_OPTS="--layout=reverse --inline-info"
  ```

  **Example: Height mode with border**
  ```bash
  export FZF_DEFAULT_OPTS="--height=40% --layout=reverse --info=inline --border --margin=1 --padding=1"
  ```

  **Example: tmux popup with fallback to height mode**
  ```bash
  # Open in tmux popup if on tmux, otherwise use --height mode
  export FZF_DEFAULT_OPTS='--height 40% --tmux bottom,40% --layout reverse --border top'
  ```

  **Example: Multi-line configuration**
  ```bash
  export FZF_DEFAULT_OPTS="
    --height 40%
    --layout reverse
    --info inline
    --border
    --margin 1
    --padding 1
  "
  ```

  <Warning>
    Do not add `--preview` option to `FZF_DEFAULT_OPTS`. Since fzf is a general-purpose text filter rather than a file finder, the preview command may not work correctly with non-file inputs.
  </Warning>
</ParamField>

<ParamField path="FZF_DEFAULT_OPTS_FILE" type="string">
  Path to a file containing default options. If you prefer to manage default options in a file rather than an environment variable, set this variable to point to the location of the file.

  ```bash
  export FZF_DEFAULT_OPTS_FILE=~/.fzfrc
  ```

  The file should contain fzf options, one per line or space-separated:

  ```text
  --height 40%
  --layout reverse
  --info inline
  --border
  ```
</ParamField>

## Shell Integration Variables

<Warning>
  `FZF_DEFAULT_COMMAND` is **not used** by shell integration due to the slight difference in requirements. Shell integration uses its own set of environment variables.
</Warning>

### CTRL-T File Selection

<ParamField path="FZF_CTRL_T_COMMAND" type="string">
  Command to run to generate the list of files and directories for CTRL-T.

  ```bash
  export FZF_CTRL_T_COMMAND="fd --type f --strip-cwd-prefix"
  ```

  To disable the CTRL-T binding, set this to an empty string when sourcing the integration script:
  ```bash
  FZF_CTRL_T_COMMAND= eval "$(fzf --bash)"
  ```
</ParamField>

<ParamField path="FZF_CTRL_T_OPTS" type="string">
  Additional options for fzf when invoked by CTRL-T.

  ```bash
  export FZF_CTRL_T_OPTS="
    --walker-skip .git,node_modules,target
    --preview 'bat -n --color=always {}'
    --bind 'ctrl-/:change-preview-window(down|hidden|)'
  "
  ```
</ParamField>

### CTRL-R History Search

<ParamField path="FZF_CTRL_R_COMMAND" type="string">
  Custom override for CTRL-R is not yet supported and will emit a warning if set to a non-empty value.

  To disable the CTRL-R binding, set this to an empty string when sourcing the integration script:
  ```bash
  FZF_CTRL_R_COMMAND= eval "$(fzf --bash)"
  ```
</ParamField>

<ParamField path="FZF_CTRL_R_OPTS" type="string">
  Additional options for fzf when invoked by CTRL-R.

  ```bash
  export FZF_CTRL_R_OPTS="
    --bind 'ctrl-y:execute-silent(echo -n {2..} | pbcopy)+abort'
    --color header:italic
    --header 'Press CTRL-Y to copy command into clipboard'
  "
  ```
</ParamField>

### ALT-C Directory Navigation

<ParamField path="FZF_ALT_C_COMMAND" type="string">
  Command to run to generate the list of directories for ALT-C.

  ```bash
  export FZF_ALT_C_COMMAND="fd --type d --strip-cwd-prefix"
  ```

  To disable the ALT-C binding, set this to an empty string when sourcing the integration script:
  ```bash
  FZF_ALT_C_COMMAND= eval "$(fzf --bash)"
  ```
</ParamField>

<ParamField path="FZF_ALT_C_OPTS" type="string">
  Additional options for fzf when invoked by ALT-C.

  ```bash
  export FZF_ALT_C_OPTS="
    --walker-skip .git,node_modules,target
    --preview 'tree -C {}'
  "
  ```
</ParamField>

## Fuzzy Completion Variables

<ParamField path="FZF_COMPLETION_TRIGGER" type="string" default="**">
  Trigger sequence for fuzzy completion.

  ```bash
  # Use ~~ as the trigger sequence instead of the default **
  export FZF_COMPLETION_TRIGGER='~~'
  ```
</ParamField>

<ParamField path="FZF_COMPLETION_OPTS" type="string">
  Options to pass to fzf for completion.

  ```bash
  export FZF_COMPLETION_OPTS='--border --info=inline'
  ```
</ParamField>

<ParamField path="FZF_COMPLETION_PATH_OPTS" type="string">
  Options for path completion (e.g., `vim **<TAB>`).

  ```bash
  export FZF_COMPLETION_PATH_OPTS='--walker file,dir,follow,hidden'
  ```
</ParamField>

<ParamField path="FZF_COMPLETION_DIR_OPTS" type="string">
  Options for directory completion (e.g., `cd **<TAB>`).

  ```bash
  export FZF_COMPLETION_DIR_OPTS='--walker dir,follow'
  ```
</ParamField>

## HTTP Server Variables

<ParamField path="FZF_API_KEY" type="string">
  X-API-Key header value for HTTP server when using `--listen` option. This provides authentication for remote connections.

  ```bash
  export FZF_API_KEY="your-secret-key-here"
  fzf --listen 6266
  ```
</ParamField>

## Other Variables

<ParamField path="NO_COLOR" type="string">
  When set to any value, disables color output. This follows the [NO_COLOR](https://no-color.org/) standard.

  ```bash
  export NO_COLOR=1
  ```
</ParamField>

<ParamField path="RUNEWIDTH_EASTASIAN" type="string">
  When set to `1`, enables East Asian Ambiguous Width handling. This affects how certain Unicode characters are displayed.

  ```bash
  export RUNEWIDTH_EASTASIAN=1
  ```
</ParamField>

## Complete Configuration Example

<Tabs>
  <Tab title="Bash/Zsh">
    ```bash
    # Default command for file listing
    export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix --hidden --follow --exclude .git'

    # Default options
    export FZF_DEFAULT_OPTS='
      --height 40%
      --tmux bottom,40%
      --layout reverse
      --border top
      --info inline
    '

    # CTRL-T configuration
    export FZF_CTRL_T_COMMAND="$FZF_DEFAULT_COMMAND"
    export FZF_CTRL_T_OPTS="
      --preview 'bat -n --color=always {}'
      --bind 'ctrl-/:change-preview-window(down|hidden|)'
    "

    # CTRL-R configuration
    export FZF_CTRL_R_OPTS="
      --preview 'echo {}'
      --preview-window up:3:hidden:wrap
      --bind 'ctrl-/:toggle-preview'
      --bind 'ctrl-y:execute-silent(echo -n {2..} | pbcopy)+abort'
      --color header:italic
      --header 'Press CTRL-Y to copy command into clipboard'
    "

    # ALT-C configuration
    export FZF_ALT_C_OPTS="
      --preview 'tree -C {} | head -200'
    "

    # Completion configuration
    export FZF_COMPLETION_TRIGGER='**'
    export FZF_COMPLETION_OPTS='--border --info=inline'
    ```
  </Tab>
  <Tab title="Fish">
    ```fish
    # Default command for file listing
    set -x FZF_DEFAULT_COMMAND 'fd --type f --strip-cwd-prefix --hidden --follow --exclude .git'

    # Default options
    set -x FZF_DEFAULT_OPTS '
      --height 40%
      --tmux bottom,40%
      --layout reverse
      --border top
      --info inline
    '

    # CTRL-T configuration
    set -x FZF_CTRL_T_COMMAND "$FZF_DEFAULT_COMMAND"
    set -x FZF_CTRL_T_OPTS "
      --preview 'bat -n --color=always {}'
      --bind 'ctrl-/:change-preview-window(down|hidden|)'
    "

    # ALT-C configuration
    set -x FZF_ALT_C_OPTS "
      --preview 'tree -C {} | head -200'
    "

    # Completion configuration (Fish-specific)
    set -x FZF_COMPLETION_OPTS '--border --info=inline'
    ```
  </Tab>
</Tabs>

<Tip>
  Display modes for shell integration bindings can be separately configured via `FZF_{CTRL_T,CTRL_R,ALT_C}_OPTS` or globally via `FZF_DEFAULT_OPTS`. For example:
  
  ```bash
  export FZF_CTRL_R_OPTS='--tmux bottom,60% --height 60% --border top'
  ```
</Tip>
